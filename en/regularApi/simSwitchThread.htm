Regular API function

sim.switchThread

Description Allows specifying the exact moment at which the current thread or coroutine should switch to another thread or yield. By default, CoppeliaSim doesn't use regular threads, but something similar to hybrid threads (which behave like coroutines). This allows much more flexibility and execution control of the threads: each thread (except for the main or application thread) has a switch timing associated, which specifies how long the thread will run before switching to other threads. By default this value is 2 millisecond, but can be modified with sim.setThreadSwitchTiming. That timing can be shortened with sim.switchThread. See also the sim.setThreadAutomaticSwitch, sim.setThreadResumeLocation and sim.setThreadIsFree functions.
